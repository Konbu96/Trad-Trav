# お役立ち情報・原稿用シート

Google スプレッドシートや Excel に **ファイル → インポート** で取り込める CSV です。種類ごとにファイルを分けています。

**列が横に長くて分かりにくい場合**は、項目の意味だけ **縦に並べた一覧** を `sheet-field-reference.md` に書いてあります。先にそちらを読むと対応が取りやすいです。

### CSV の形の読み方

- **1行目**: 列の名前（ヘッダー）。左から順に「項目1」「項目2」…と並んでいます。
- **2行目**: 記入例（消して差し替えてよい）。
- **3行目以降**: 1行につき、マナー1件またはトピック1件。

| ファイル | 内容 | アプリへの反映先（概要） |
|----------|------|-------------------------|
| `manner-items-template.csv` | マナー（項目）の列見本 | **列 A〜G** の定義例。実データはリポジトリ内の **`マナー情報 - マナー情報.csv`** を正とし、取り込み先は `app/data/manners.ts` の `MANNER_ITEMS` |
| `豆知識 - 豆知識.csv` | 豆知識（統合タブ用） | 列は `id` / `タイトル` / `本文` / `ポイント` / `キーワード` / `シーン`。取り込み先は `app/data/helpfulInfo.ts` の `TIPS_TOPICS` と `app/i18n/sheetTipsTopics.ts`（各言語の topics） |
| `trivia-topics-template.csv` | 豆知識系（統合タブ用） | `app/data/helpfulInfo.ts` の `TIPS_TOPICS` の一部として取り込み、`tabId` は `tips` |
| `travel-guide-topics-template.csv` | 旅ガイド系（統合タブ用） | 同上（豆知識と旅ガイドはアプリ上は1タブ「豆知識・旅ガイド」に統合） |

## 共通ルール

- **文字コード**: UTF-8
- **1行 = 1項目（1トピック）**

### マナー用シート（A〜G）のルール

| 列 | 見出し | ルール |
|:--:|--------|--------|
| A | `id` | 英語スネークケース推奨（例 `meal-before-greeting`）。`MANNER_ITEMS[].id` と一致。 |
| B | `カテゴリ` | `食事` / `観光` / `移動` / `生活` / `ルール`（日本語）または `meal` / `sightseeing` / `mobility` / `daily` / `rules`（コードと同じ英語）。 |
| C | `見出し` | `title` に対応。 |
| D | `本文` | `shortDescription` に対応。 |
| E | `ポイント` | 複数は **セミコロン `;`** で区切り → コードでは `details: ["…", "…"]`。 |
| F | `キーワード` | **セミコロン `;`** → `keywords: ["…", "…"]`。 |
| G | `シーン` | トークンを **セミコロン `;`** で区切る（例 `facility;walking`）→ `scenes`。使える値は `sheet-field-reference.md` 参照。 |

**「表に書いているだけ」で大丈夫か**  
大丈夫です。**1行＝マナー1件**で、A〜G はそれぞれ **その行の1セル** という前提です。セル内で改行して複数行にしても構いません（スプレッドシートでは Alt+Enter／Excel では同様）。

**`;` の話は何を変えるか**  
アプリのプログラムを変える話ではありません。**`manners.ts` に手で写すとき**に、`details` / `keywords` / `scenes` が **JavaScript の配列**になるようにするための目安です。

- **ポイント（E）**が複数ある → コードでは `details: ["1行目", "2行目"]`。シートでは **同じセル内**を `;` で区切るか、**改行**で区切る。写すときに `;` または改行のところで文字列を分けて配列にする。
- **キーワード（F）**・**シーン（G）**も同様。複数なら1セル内を `;` で区切るのが CSV 出力時に壊れにくい、という**おすすめ**です。すでに1語だけなら、そのまま1要素の配列でよいです。

**列を増やしてポイントを H列・I列…に分けた**場合は、このリポジトリの A〜G テンプレとは別設計なので、コードに写すときだけ1つの `details` 配列にまとめてください。

### マナー: シートとアプリの表示が一致するようにするには

アプリは Google シートに **自動接続していません**。シートで編集したあと、次の **いずれか** をコード側に反映してください。

1. **日本語の見出し・本文・ポイント・カテゴリ・シーン・キーワード（A〜G）**  
   → `app/data/manners.ts` の `MANNER_ITEMS` を、シートの行と対応させて更新する（`カテゴリ` は上表のとおり `categoryId` に変換）。  
   → 表示言語が日本語のときは、この `MANNER_ITEMS` がそのまま使われます（`helpfulLibrary` の日本語 `mannerItems` は上書きされません）。

2. **英語 UI**  
   → マナー用 A〜G シートには英語列がありません。`app/i18n/helpfulLibraryContent.ts` の英語 `mannerItems` を、同じ `id` で別途更新する。未登録の項目は日本語（`MANNER_ITEMS`）にフォールバックします。

3. **中国語・韓国語**  
   → `app/i18n/helpfulLibraryZh.ts` / `helpfulLibraryKo.ts` の `mannerItems` を同様に更新する（パッチは `localePatches/` も参照）。

### 日本語と英語（マナー）

- **見出し・本文・ポイント**は A〜G シートの日本語（マナーは **`manners.ts` の `MANNER_ITEMS` に反映**するのが正本）。
- **英語**は `helpfulLibraryEn.mannerItems` に同じ `id` で登録する（マナー用オリジナルシートの列外）。

## サンプル行

各 CSV の2行目は既存データの例です。**編集用に複製して使うか、削除してから**自分の行だけにしてください。
